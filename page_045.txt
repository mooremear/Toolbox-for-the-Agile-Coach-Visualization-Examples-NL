Taken < 1 dag

Stop bij het maken van taken (dus het opsplitsen van het werk dat gedaan moet worden om een User Story af te krijgen), niet tot alle taken minder dan een dag kosten om afgerond te worden.

Deze regel maakt het eenvoudiger om voortgang en eventuele bottlenecks te zien. Het geeft ook aanleiding voor dieper inhoudelijke discussies tijdens de planning sessies.

Verder zie je, als je Dotting gebruikt, het snel als er iets niet in de haak is doordat een taak punten verzamelt. Geen enkele taak zou immers meer dan 1 stip mogen krijgen.

Als je een Sprint Burndown gebruikt, kun je in plaats van het tellen van uren ook gewoon het aantal taken tellen. Veel sneller!

Task size < 1 day

When tasking (i.e. breaking down the work that needs to be performed to complete a User Story) donâ€™t stop splitting the tasks until each task can be completed within a day.

This policy makes it easier to see progress and bottlenecks. It also triggers deeper collaborative discussions during the planning sessions.

Furthermore, if you use Dotting, you know there is something fishy going on if a task starts collecting dots since no single task should have more than one dot.

If you use a Sprint Burndown, you can instead of summarizing hours summarize the number of tasks. Much faster!
